# BVE Dashboards for Agentic AI Coding

Measures the efficiency and quality impact of **GitHub Copilot Coding Agent** (autonomous agent that creates PRs).

## Overview

The Agentic AI Coding dashboard focuses on **autonomous coding sessions** where the agent:
- Receives an issue/PR assignment
- Works independently to generate code
- Creates a pull request
- Iterates based on feedback

This is different from AI Assisted Coding (IDE features) - here we measure **autonomous agent productivity**.

## Quick Start

### View Dashboard

```bash
open dashboard/efficiency/index.html
```

### Collect Data

```bash
# Set credentials
export GITHUB_TOKEN="your_token_here"

# Run query script
cd data/queries
ENTERPRISE="my-enterprise" DAYS=28 ./agentic-ai-coding.sh > my-data.json
```

### Run Tests

```bash
cd dashboard/efficiency
npm install --include=dev
npm test
```

## Key Metrics

### Impact Section
- **Total Human-Equivalent Hours Saved** - Time that would have been spent manually
- **Org Hrs / Day** - Daily organizational savings
- **Hrs / Human Dev / Day** - Per-developer benefit
- **Economic Value** - Labor cost savings

### Drivers Section
- **Human Devs with Agent Activity** - Developers using the agent
- **Agent Sessions Started** - Total autonomous coding sessions
- **PRs Created by Agent** - Pull requests authored by agent
- **PRs Merged** - Successfully integrated changes
- **Merged LoC** - Lines of code in merged PRs
- **Turbulence** - Friction signals (rework, confusion, quality issues)

### Quality Section
- **Merge Rate** - % of agent PRs that get merged
- **Session Duration Distribution** - Time spent per session
- **LoC per Session** - Code generated per session
- **PR Details Table** - Individual PR tracking

## Estimation Methods

### 1. Duration-Based (Default)

**Formula:** `(merged_session_minutes / 60) × human_to_agent_ratio`

**Default Ratio:** 10× (agent works 10× faster than human)

**Only counts MERGED PRs** - Unsuccessful PRs don't save time

**Example:**
- 60 minutes of agent time (merged PR)
- × 10 human-equivalent multiplier
- = 10 human-equivalent hours saved

### 2. LoC-Based

**Formula:** `(merged_loc_added / 1000) × hours_per_kloc`

**Default Rate:** 4 hours per 1,000 lines (vs 2 for AI assisted)

**Higher rate because:** Agent generates more complete, complex code

**Example:**
- 2,000 merged LoC
- / 1,000 × 4
- = 8 hours saved

## Key Differences from AI Assisted Coding

| Aspect | AI Assisted | Agentic |
|--------|-------------|---------|
| **Scope** | IDE features (completion, chat) | Autonomous agent sessions |
| **Counting** | All interactions | Only MERGED PRs |
| **LoC Rate** | 2 hrs/kloc | 4 hrs/kloc |
| **Metric** | Interactions, acceptances | Sessions, PRs, turbulence |
| **Focus** | Individual developer velocity | Agent autonomous productivity |

## Data Sources

### Required Data
Generated by `agentic-ai-coding.sh`:
- `developer_day_summary` - Per-developer-per-day activity
- `pr_sessions` - PR details with duration, changes, review metrics
- `requests` - Issue/PR assignment history

### Optional Configuration
- `est_human_to_agent_ratio` (default: 10)
- `est_hrs_per_kloc` (default: 4)
- `cfg_labor_cost_per_hour` (default: 150)

## Turbulence Metrics

**Turbulence** measures friction/quality issues:

1. **Repeat Assignments** - Same issue assigned multiple times
2. **Closed Without Merge** - PRs abandoned
3. **Zero Duration Sessions** - Failed/aborted sessions
4. **Zero Changes** - PRs with no code
5. **Review Friction** - Changes requested, reopens, guidance needed
6. **Extra Invocations** - Follow-up requests beyond initial

**High turbulence** = Agent struggling, needing guidance, or producing low-quality output

## Quality Signals

### Good Quality
- ✅ High merge rate (>70%)
- ✅ Low turbulence (<3 per PR)
- ✅ Consistent session durations
- ✅ Meaningful LoC per session

### Quality Concerns
- ⚠️ Low merge rate (<50%)
- ⚠️ High turbulence (>5 per PR)
- ⚠️ Many zero-duration sessions
- ⚠️ PRs with zero changes
