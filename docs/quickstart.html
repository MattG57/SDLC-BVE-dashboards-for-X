Quick Start Guide

How to collect data and load it into each dashboard.

Prerequisites

  • gh CLI installed and authenticated (gh auth login)
  • A GitHub personal access token with the required scopes:
    • copilot — Copilot metrics access
    • read:org or read:enterprise — organization/enterprise data
# Export your token export GITHUB_TOKEN="your_token_here"

AI Assisted Coding Dashboards

The Efficiency and Structural dashboards share the same data source — a JSON export from the AI-assisted coding query script.

1

Collect data

Run the query script from the repository root. Use ENTERPRISE or ORG to target your scope.

# Enterprise-wide metrics (last 28 days) ENTERPRISE="my-enterprise" DAYS=28 \ ./BVE-dashboards-for-ai-assisted-coding/data/queries/ai-assisted-coding.sh > my-data.json # Organization-level metrics ORG="my-org" DAYS=28 \ ./BVE-dashboards-for-ai-assisted-coding/data/queries/ai-assisted-coding.sh > my-data.json
⚠️

Do not redirect stderr into your JSON file. The script writes progress to stderr and data to stdout.
✅ ./script.sh > data.json
❌ ./script.sh > data.json 2>&1

2

Open a dashboard

Open the desired dashboard HTML file directly in your browser.

# Efficiency Dashboard open BVE-dashboards-for-ai-assisted-coding/dashboard/efficiency/index.html # Structural Dashboard open BVE-dashboards-for-ai-assisted-coding/dashboard/structural/index.html
3

Upload your data

Use the upload area in the dashboard to load your my-data.json file. The dashboard processes everything client-side — no data leaves your browser.

Configuration

You can customize estimation constants via the dashboard UI or by uploading a separate config JSON:

ParameterDefaultDescription
est_interactions_per_hour20Interactions per developer-hour (interactions method)
est_hrs_per_kloc2Hours per 1,000 lines of code (LoC method)
cfg_labor_cost_per_hour$150Fully-loaded labor cost
cfg_baseline_hrs_per_dev_per_week30Baseline coding hours per developer per week
cfg_workdays_per_week5Working days per week

Estimation Methods

Efficiency Dashboard

Translates Copilot telemetry into hours saved and labor cost avoided using three approaches:

  • Interactions-Based — interactions / est_interactions_per_hour
  • LoC-Based — (loc_added / 1000) × est_hrs_per_kloc
  • Manual Daily % — Configured percentage of baseline developer hours

Structural Dashboard

Assesses whether the structural conditions for productive AI-assisted coding exist:

  • Adoption Completeness — Is Copilot spreading across teams?
  • Usage Consistency — Is usage steady or volatile?
  • Output Correlation — Does higher adoption correlate with more output?

Agentic AI Coding Dashboard

Measures the impact of GitHub Copilot Coding Agent — the autonomous agent that creates pull requests.

1

Collect data

Run the agentic query script. You can target an entire org or a single repository.

# All repositories in an org (may take 10-15 minutes for large orgs) ORG="my-org" DAYS=28 \ ./BVE-dashboards-for-agentic-ai-coding/data/queries/agentic-ai-coding.sh > agentic-data.json # Single repository (faster) ORG="my-org" REPO="my-repo" DAYS=28 \ ./BVE-dashboards-for-agentic-ai-coding/data/queries/agentic-ai-coding.sh > agentic-data.json
2

Open the dashboard

open BVE-dashboards-for-agentic-ai-coding/dashboard/efficiency/index.html
3

Upload your data

Load agentic-data.json via the browser upload area. All processing is client-side.

Configuration

ParameterDefaultDescription
est_human_to_agent_ratio10Human-to-agent time multiplier (duration method)
est_hrs_per_kloc4Hours per 1,000 lines of code (higher than IDE — agent generates more complex code)
cfg_labor_cost_per_hour$150Fully-loaded labor cost

Estimation Methods

Duration-Based (Default)

(merged_session_minutes / 60) × human_to_agent_ratio

Only counts merged PRs. The agent completes in minutes what would take a human hours — this method applies a 10× multiplier to agent time.

LoC-Based

(merged_loc_added / 1000) × hours_per_kloc

Uses a rate of 4 hrs/kloc (vs 2 for IDE-assisted) because the agent generates more complete, standalone code.

💡

Key difference from AI Assisted Coding: The agentic dashboard only counts merged PRs. Unsuccessful agent PRs are not counted as time saved but are tracked as turbulence signals.

Verify Calculations

You can verify that dashboard math is correct using the included verification scripts:

# AI Assisted Coding verification cd BVE-dashboards-for-ai-assisted-coding/data node verify_math.cjs

This runs example data through all estimation methods and outputs raw values, calculated hours, and per-developer averages.
